# 文档编写规范

本文档定义了LaTeX模板工程各仓库用户文档的编写规范。

## 文件命名

每个`.sty`或`.cls`文件对应一个同名的`.md`文件，放置在同一目录下。

如`minimus-math.sty`对应`minimus-math.md`。

## 文档结构

文档由一级标题`# 包名`开头，然后依次是三个二级标题。

```
## 引入的宏包
## 自定义命令
## 自定义环境
```

## 标题规范

### 二级标题

固定使用上述三个二级标题。

### 三级标题

- 每个引入的宏包、自定义命令、自定义环境单独一个三级标题
- 标题仅包含名称，用`` ` ``包裹，不附带描述和参数格式
- 相似命令可合并到同一个标题，但每个命令名都必须出现在标题中
- 示例：`` ### `\N`、`\Z`、`\Q`、`\R`、`\C`、`\emptyset` ``
- 环境标题不含`\begin`/`\end`，仅写环境名：`` ### `Equation` ``
- 包标题仅写包名：`` ### `amsmath`、`amsthm`、`amssymb` ``

### 三级标题后必须跟一行简要说明

用一句话概括用途，允许与下文表格内容有重复。

## 表格规范

### 每个三级标题下必须有表格

宏包用宏包表格（两列`宏包`、`说明`），命令用命令表格（两列`命令`、`说明`），即便只有一个条目也要有表格。

命令表格中有参命令在命令列带参数。参数使用英文，如`` `\Sum[sub][sup]` ``。说明列用内联`$...$`写数学输出。

### 宏包的预配置也须用表格

宏包标题下如有`\PassOptionsToPackage`、`\ctikzset`、`\lstset`等预配置项，须用两列表格（`选项`、`说明`）列出。值与选项写在同一列（如`resistors/scale=0.65`）。禁止使用列表形式。

### 有参命令和环境附加参数表格

四列：`序号`、`格式`、`类型`、`说明`

| 列 | 说明 |
|----|------|
| 序号 | 参数顺序 |
| 格式 | 实际书写形式，如`[label]`、`{content}`、`*`、`!` |
| 类型 | 必选、可选、标记、分隔 |
| 说明 | 参数含义及默认值 |

## 环境签名

环境在参数表格之前需要一个代码块签名，展示完整调用形式。

签名之后是参数表格，之后按需给出示例代码块。签名和示例的代码块内全部使用英文。

## 语言规范

- 中英文间不加空格
- 代码内参数使用英文：`\Int[low][high]`而非`\Int[下标][上标]`
- 代码块（`` ``` ``或`` ` ``）内的内容全部使用英文
- 禁止Unicode箭头符号`→`，用文字替代（如"产生"、"输出"）

## 数学符号规范

内联`$...$`数学模式中不得使用如下包的命令（在纯Markdown渲染环境中不可用）：

- `physics`：`\dd`、`\vb`、`\vb*`等
- `upgreek`：`\uppsi`、`\updelta`等
- `accents`：`\underaccent`等

替代方案：使用标准LaTeX命令（`\mathrm`、`\mathbf`、`\mathbb`、`\operatorname`等），或改用文字描述。
