# 编写规范

本文档定义LaTeX模板工程各仓库用户文档的编写规范。

## 文件命名

- 宏包（`.sty`）：每个`.sty`对应一个同名的`.md`文件，放置在同一目录下。如`minimus-math.sty`对应`minimus-math.md`。
- 文档类（`.cls`）：文档直接写在`README.md`中。

## 语言规范

- 中英文间不加空格
- 代码内参数使用英文：`\Int[low][high]`而非`\Int[下标][上标]`
- 代码块（`` ``` ``或`` ` ``）内的内容全部使用英文
- 禁止Unicode箭头符号`→`，用文字替代（如"产生"、"输出"）

## 数学规范

内联`$...$`数学模式中不得使用以下包的命令（在纯Markdown渲染环境中不可用）：

- `physics`：`\dd`、`\vb`、`\vb*`等
- `upgreek`：`\uppsi`、`\updelta`等
- `accents`：`\underaccent`等

替代方案：使用标准LaTeX命令（`\mathrm`、`\mathbf`、`\mathbb`、`\operatorname`等），或改用文字描述。

## 标题规范

三级标题仅包含名称，用`` ` ``包裹，不附带描述和参数格式。相似命令可合并到同一个标题，但每个命令名都必须出现在标题中。环境标题不含`\begin`/`\end`，仅写环境名。包标题仅写包名。

示例：

- 命令：`` ### `\N`、`\Z`、`\Q`、`\R`、`\C`、`\emptyset` ``
- 环境：`` ### `Equation` ``
- 包：`` ### `amsmath`、`amsthm`、`amssymb` ``

每个三级标题后必须跟一行简要说明，用一句话概括用途。

## 条目规范

每个三级标题下必须有表格，即便只有一个条目。

- 引入的宏包用两列（`宏包`、`说明`）
- 自定义命令用两列（`命令`、`说明`）。有参命令在命令列带参数，参数使用英文（如`` `\Sum[sub][sup]` ``）。说明列采用"中文描述+数学符号"格式（如"自然常数$\mathrm{e}$"）。
- 文档选项用三列（`选项`、`默认值`、`说明`），注明默认值和可选值。

宏包标题下若有预配置项（`\PassOptionsToPackage`、`\ctikzset`、`\lstset`、`\tcbset`、`\sisetup`等），须用两列表格（`选项`、`说明`）列出。值与选项写在同一列（如`resistors/scale=0.65`）。禁止使用列表形式。

有参命令和环境附加四列参数表格：

| 列 | 说明 |
|----|------|
| 序号 | 参数顺序 |
| 格式 | 实际书写形式，如`[label]`、`{content}`、`*`、`!` |
| 类型 | 必选、可选、标记、分隔 |
| 说明 | 参数含义及默认值 |

环境在参数表格之前需要一个代码块签名。签名中仅用`...`占位，不含示例内容。签名之后是参数表格，之后按需给出示例代码块。示例内容应具有一致性和系列性。

```

### 文档类（`.cls`）

```

## 文档选项

## 自定义命令

```

#### 文件头

文档类的第一句采用统一格式："XXX提供了一个适用于...的文档类`xxx`，以Git子模块方式引入，基于`xxx`和`xxx`。"

命名来源单独一行说明。

#### 参见链接

在第一段之后，以纯文本URL依次注明配套Template和Minimus的链接（如"参见NotebookNeonTemplate：https://..."）。若无需Template可不写Template链接，若无需Minimus可不写Minimus链接。

#### 主要特点

以列表形式列出文档类的主要特性。写作时可灵活发挥，不要求与其他Class一致。

### 引入方式

文档类须包含引入方式小节：Git子模块命令、`\input@path`配置、`\documentclass`声明。

### 最小示例

若文档类无独立Template但有典型使用场景（如StandaloneSilicon搭配Minimus），可在引入方式之后给出一个完整的最小示例。有Template的Class不写此节（示例属于Template文档）。
