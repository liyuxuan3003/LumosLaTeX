# 编写规范（文档类）

本文档定义`.cls`文档类的用户文档编写规范。

## 文件命名

每个`.cls`对应一个同名的`.md`文件，放置在同一目录下。如`notebook-neon.cls`对应`notebook-neon.md`。

## 文档结构

文档由一级标题`# 类名`开头，然后依次是以下几个小节。

```
## 文档选项
## 引入方式
## 自定义命令
## 自定义环境
```

若该小节无内容则省略。

### 文档选项

用两列表格（`选项`、`说明`）列出`\DeclareStringOption`和`\DeclareBoolOption`等定义的键值选项，注明默认值和可选值。

### 引入方式

给出Git子模块引入命令、`\input@path`配置和`\documentclass`声明。不写完整的MWE（MWE属于Template文档）。

若存在配套的Template和推荐的Minimus，以纯文本URL形式注明（如"参见NotebookNeonTemplate：https://..."）。

## 标题规范

三级标题仅包含名称，用`` ` ``包裹，不附带描述和参数格式。相似命令可合并到同一个标题，但每个命令名都必须出现在标题中。环境标题不含`\begin`/`\end`，仅写环境名。

示例：

- 命令：`` ### `\N`、`\Z`、`\Q`、`\R`、`\C`、`\emptyset` ``
- 环境：`` ### `Equation` ``

每个三级标题后必须跟一行简要说明。

## 表格规范

每个三级标题下必须有表格，即便只有一个条目。

- 自定义命令用两列（`命令`、`说明`）。有参命令在命令列带参数，参数使用英文。说明列采用"中文描述+数学符号"格式。
- 自定义环境用环境表格或用参数表格+代码块签名。

## 参数表格

有参命令和环境附加四列参数表格：

| 列 | 说明 |
|----|------|
| 序号 | 参数顺序 |
| 格式 | 实际书写形式，如`[label]`、`{content}`、`*`、`!` |
| 类型 | 必选、可选、标记、分隔 |
| 说明 | 参数含义及默认值 |

## 环境

环境在参数表格之前需要一个代码块签名。签名中仅用`...`占位，不含示例内容。

签名之后是参数表格，之后按需给出示例代码块。示例内容应具有一致性和系列性。

## 语言规范

- 中英文间不加空格
- 代码内参数使用英文：`\Int[low][high]`而非`\Int[下标][上标]`
- 代码块（`` ``` ``或`` ` ``）内的内容全部使用英文
- 禁止Unicode箭头符号`→`，用文字替代（如"产生"、"输出"）

## 数学符号规范

内联`$...$`数学模式中不得使用以下包的命令（在纯Markdown渲染环境中不可用）：

- `physics`：`\dd`、`\vb`、`\vb*`等
- `upgreek`：`\uppsi`、`\updelta`等
- `accents`：`\underaccent`等

替代方案：使用标准LaTeX命令（`\mathrm`、`\mathbf`、`\mathbb`、`\operatorname`等），或改用文字描述。
